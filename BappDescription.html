Interactsh Collaborator integrates the Interactsh out-of-band (OOB) interaction server into Burp Suite, giving testers a dedicated tab for generating unique OOB payloads and monitoring incoming interactions across DNS, HTTP/HTTPS, SMTP, FTP, LDAP, and SMB protocols. This can be used alongside Burp's native Collaborator server for OOB testing.

Features

  • Generates unique Interactsh payload URLs, available via a one-click copy button or the right-click context menu anywhere in Burp
  • Monitors interactions in real-time with configurable polling intervals, displaying each result in a sortable, filterable log table
  • Protocol-aware detail views: HTTP/HTTPS interactions render in Burp's native request and response editors; DNS, SMTP, FTP, LDAP, and SMB interactions display formatted, structured text
  • Highlights unread interactions in bold and shows an unread count on the Interactsh tab
  • Supports self-hosted or alternative Interactsh servers, configurable with custom hostname, port, TLS toggle, and authorization token
  • Sessions use RSA-2048 key exchange and AES-encrypted polling, matching the Interactsh protocol

Usage

  1. Navigate to the "Interactsh" tab and click "Copy URL to clipboard" to copy the current payload URL, or right-click anywhere in Burp and select "Copy Interactsh URL".
  2. Inject the URL into a parameter, header, or other input in your test target and trigger a request.
  3. Watch the Logs table for incoming interactions; click a row to view full details. Use the filter buttons (All, HTTP, DNS, SMTP, LDAP, SMB, FTP) to focus on a specific protocol.
  4. To start a fresh session, click "Regenerate Interactsh Session". To poll immediately rather than waiting for the next interval, click "Refresh".
  5. To use a self-hosted server, open the "Configuration" tab, enter your server hostname, port, and authorization token, enable or disable TLS, then click "Update Settings".
